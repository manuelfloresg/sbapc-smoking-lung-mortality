# From smoking scenarios to lung cancer mortality: Sequential Bayesian APC projections

Replication repository for:

> Flores, Manuel. *From smoking scenarios to lung cancer mortality: Sequential Bayesian APC projections.*

This repository contains the code and lightweight replication outputs for the
simulation study and Uruguay empirical application reported in the manuscript
and supplementary material.

## Repository Layout

```text
Estimacion4/
  R/                         Core SBAPC model code
  adapters/                  Simulation and empirical input adapters
  runs/                      Reproduction scripts and product builders
data/
  analysis_ready/            Public analysis-ready inputs, if redistributable
  metadata/                  Input provenance and redistribution notes
output/
  figures/                   Manuscript and supplement figure exports
  tables/                    Manuscript and supplement table exports
scripts/
  check_inputs.R             Checks empirical input availability
  install_dependencies.R     Installs required R packages
  reproduce_simulations.R    Runs Section 4 / Appendix C simulation pipeline
  reproduce_uruguay.R        Runs Section 5 / Appendix D empirical pipeline
```

Heavy generated objects, local R libraries, INLA temporary folders, and private
raw data are intentionally excluded from Git.

## Software

The replication was developed under:

- Windows 11
- R 4.6.x
- INLA 25.10.19

The required R packages are listed in `requirements.R`. To install them:

```r
source("scripts/install_dependencies.R")
```

INLA is installed from the INLA repository rather than CRAN.

## Temporary Files and INLA Safety

R-INLA can create very large temporary folders. The runtime redirects temporary
files away from the project and away from Dropbox. By default, scripts use:

```text
C:/tmp_inla
```

You can override this before running:

```r
Sys.setenv(BAPC_INLA_TMPDIR = "D:/tmp_inla")
```

Do not set this path inside a synced Dropbox/OneDrive folder.

## Reproducing the Simulation Study

From the repository root:

```r
source("scripts/reproduce_simulations.R")
```

By default this runs the final 200-seed simulation configuration and writes
outputs under:

```text
Estimacion4/results/20260521_FINAL_200SEEDS/
```

For a quicker smoke run:

```r
Sys.setenv(BAPC_N_SEEDS = "2")
Sys.setenv(BAPC_OUT_BASE = "results/smoke_simulation")
source("scripts/reproduce_simulations.R")
```

The simulation pipeline is self-contained and does not require Uruguay empirical
data.

## Reproducing the Uruguay Empirical Application

From the repository root:

```r
source("scripts/check_inputs.R")
source("scripts/reproduce_uruguay.R")
```

The empirical pipeline requires analysis-ready Uruguay inputs. Public defaults
look for these files under `data/analysis_ready/`:

- `uruguay_mortality_smooth_cancer.csv`
- `uruguay_population_1950_2070.dta`
- `uruguay_smoking_prevalence_aggregated.csv`
- `uruguay_incidence_smooth_1998_2022.csv`

If the files are stored elsewhere, set:

```r
Sys.setenv(BAPC_PATH_MORT_CSV = "path/to/uruguay_mortality_smooth_cancer.csv")
Sys.setenv(BAPC_PATH_POP_DTA  = "path/to/uruguay_population_1950_2070.dta")
Sys.setenv(BAPC_PATH_PREV_DTA = "path/to/uruguay_smoking_prevalence_aggregated.csv")
Sys.setenv(BAPC_PATH_INC_CSV  = "path/to/uruguay_incidence_smooth_1998_2022.csv")
```

Empirical outputs are written under:

```text
Estimacion4/results/20260520_URUGUAY_CANDIDATE/
```

## Data Availability and Redistribution

Some source files used in the Uruguay application were obtained from Uruguayan
institutions, including the Ministry of Public Health and the National Cancer
Registry. Raw source files are not redistributed. The repository includes
analysis-ready derived inputs that are non-identifiable and suitable for
redistribution. Provenance and redistribution status are documented in
`data/metadata/`.

See `DATA_AVAILABILITY.md` for the complete replication data note.

## Approximate Runtime

Runtime is hardware dependent. The 200-seed simulation is computationally
expensive and should be treated as an overnight or longer run on a workstation.
The empirical Uruguay run is shorter but still requires repeated INLA fits.

For reviewer checks, use the 2-seed smoke run described above.
